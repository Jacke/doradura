# Dora Welcome Menu -- 10 Variants

Current message key: `commands.no_links` in `locales/ru/main.ftl` (line 123)

All variants use MarkdownV2 escaping conventions for Telegram.
Note: `❤️‍🔥` is the bot's signature emoji, keep it.

---

## VARIANT 1 -- Ultra-minimal (one-liner hook)

```
Привет! Я Дора ❤️‍🔥

Кинь ссылку — скачаю. Кинь файл — конвертирую.

YouTube · TikTok · Instagram · Spotify · SoundCloud и ещё 1000+ сайтов
```

**Psychology:** Cognitive load reduction. The brain processes 3 short lines faster than a wall of features. The "1000+ сайтов" creates scale impression without listing anything. The two symmetrical "Кинь X — сделаю Y" phrases use parallelism for instant comprehension and memorability. Users who already know what they want (most of them) get zero friction -- just paste a link.

## VARIANT 2 -- Ultra-minimal (mystery + number)

```
Я Дора. Скачиваю всё ❤️‍🔥

Ссылка → видео, музыка, субтитры
Файл → конвертация в любой формат

/settings — настроить качество и формат
```

**Psychology:** Curiosity gap through vagueness ("всё"). The arrow notation creates a visual input/output mental model. Minimal but includes the one CTA that matters (settings), reducing the "now what?" moment. The word "любой" triggers completeness bias -- users assume it really does handle everything.

## VARIANT 3 -- Benefit-focused (what you GET)

```
Привет! Я Дора ❤️‍🔥

🎵 Музыка в 320 kbps за секунды
📹 Видео до 4K с субтитрами
📱 Рингтон для телефона из любого трека
✂️ Нужен только фрагмент? Добавь время: url 1:00-2:30

Просто отправь ссылку — остальное сделаю сама.
```

**Psychology:** Benefit-first copywriting. Each line answers "what's in it for me?" rather than describing features. "320 kbps", "4K", "за секунды" are concrete proof points that build credibility. The closing line removes all decision paralysis -- user doesn't need to choose anything, just send a link. The ringtone line plants a seed of a use case the user might not have considered (expansion of perceived value).

## VARIANT 4 -- Benefit-focused (problem/solution pairs)

```
Привет! Я Дора ❤️‍🔥

Хочешь трек из YouTube? Кинь ссылку — получишь MP3.
Нужно видео без водяных знаков? Кинь ссылку — скачаю в оригинальном качестве.
Есть DOCX, а нужен PDF? Пришли файл — конвертирую.

Я работаю с YouTube, TikTok, Instagram, Spotify, SoundCloud и ещё 1000+ сайтов.

Попробуй прямо сейчас — отправь ссылку!
```

**Psychology:** Problem-agitation-solution framework adapted to microcopy. Each line starts with a relatable pain point ("Хочешь...", "Нужно...", "Есть X, а нужен Y?"), creating instant recognition. The user sees themselves in at least one scenario, which triggers the endowment effect -- they already mentally "own" the solution. "Без водяных знаков" addresses a specific frustration that builds trust (the bot understands real problems).

## VARIANT 5 -- Action-oriented / interactive (direct commands)

```
Я Дора — твой медиа-ассистент ❤️‍🔥

Попробуй прямо сейчас:

1. Скопируй ссылку с YouTube или TikTok
2. Вставь сюда
3. Готово!

Или пришли любой файл — конвертирую.

Хочешь больше? Жми /settings
```

**Psychology:** Instructional design + Zeigarnik effect. Numbered steps create a micro-task that feels almost complete before the user even starts ("just 3 steps!"). The brain wants to finish what it started. "Попробуй прямо сейчас" is an activation prompt -- it shifts the user from passive reading to active doing. The "Хочешь больше?" creates a curiosity loop that drives deeper exploration only after the first action.

## VARIANT 6 -- Action-oriented (challenge/dare)

```
Привет! Я Дора ❤️‍🔥

Скинь мне любую ссылку — спорим, скачаю?

YouTube, TikTok, Instagram, Spotify, SoundCloud, VK, Twitch... и ещё 1000+ сайтов.

Видео · Музыка · Субтитры · Рингтоны · Конвертация

Ну что, проверим? Кидай ссылку!
```

**Psychology:** Challenge framing triggers reactance -- when someone dares you, you want to test them. "Спорим, скачаю?" is a low-stakes social contract that makes the first interaction feel like a game, not a transaction. The ellipsis after site names implies the list is even longer (anchoring effect). The closing "Ну что, проверим?" creates urgency without pressure -- it's an invitation disguised as a dare.

## VARIANT 7 -- Personality/fun (casual friend)

```
Хэй! Я Дора ❤️‍🔥

Короче, я скачиваю всё из интернета.
YouTube, TikTok, Insta, Spotify — вообще без разницы.

Видео до 4K, музыку в MP3/FLAC, рингтоны, субтитры — что угодно.
Могу вырезать кусочек, наложить басс-буст, сконвертить файл.

Не надо ничего настраивать — просто кинь ссылку.
А если хочешь покопаться — /settings
```

**Psychology:** Parasocial relationship building through casual register. "Короче", "вообще без разницы", "покопаться" -- this is how a tech-savvy friend explains things, not a product page. The contrast between "не надо ничего настраивать" (low barrier) and "если хочешь покопаться" (depth available) satisfies both novice and power users without overwhelming either. The casual tone reduces perceived effort -- if the bot sounds chill, using it must be easy.

## VARIANT 8 -- Personality/fun (confident and playful)

```
Привет! Я Дора ❤️‍🔥

Что я делаю? Скачиваю то, что ты не можешь скачать.
YouTube, TikTok, Insta — видео, музыку, даже субтитры.

А ещё:
— вырезаю нужный момент
— делаю рингтоны
— конвертирую файлы
— качаю в 4K, если хочешь

Просто кинь ссылку. Я разберусь.
```

**Psychology:** Self-efficacy framing ("то, что ты не можешь скачать") positions the bot as an empowering tool, not just a downloader. The confident "Я разберусь" removes cognitive burden -- the user doesn't need to know how it works. The em-dash list is visually lighter than emoji-heavy trees and reads like spoken language. "Даже субтитры" uses the surprise element to make a niche feature feel like a bonus, not noise.

## VARIANT 9 -- Progressive disclosure (minimal + inline buttons)

```
Привет! Я Дора — скачиваю и конвертирую медиа ❤️‍🔥

Отправь ссылку или файл — я сделаю остальное.
```

**Inline buttons below the message:**
`[ 🎵 Что умею ]  [ ⚙️ Настройки ]  [ 💎 Подписка ]`

Tapping "Что умею" expands to the full capabilities list (existing `menu.services_text`).

**Psychology:** Progressive disclosure -- show the minimum viable information upfront, reveal details on demand. Research shows that 60-80% of users never read past the first 3 lines of a welcome message. This variant respects that by making the default experience frictionless (paste link, get file) while offering depth for the curious. Inline buttons create affordance -- the user knows more info exists but isn't forced to process it.

## VARIANT 10 -- Progressive disclosure (tiered reveal)

```
Я Дора ❤️‍🔥 Скачиваю медиа из 1000+ сайтов.

Кинь ссылку — получишь файл.
```

**Inline buttons (row 1):**
`[ 🎵 Музыка ]  [ 📹 Видео ]  [ 🔄 Конвертация ]`

**Inline buttons (row 2):**
`[ ⚙️ Настройки ]  [ 💎 Подписка ]`

Each button opens a focused mini-card:

**"Музыка" card:**
```
🎵 Музыка и аудио

• YouTube, Spotify, SoundCloud, Bandcamp
• MP3 до 320 kbps · FLAC · WAV
• Вырезка фрагментов, аудиоэффекты
• Рингтоны для iPhone и Android

Отправь ссылку на трек — скачаю!
```

**"Видео" card:**
```
📹 Видео

• YouTube, TikTok, Instagram, Twitch, VK
• Качество до 4K · субтитры
• Кружки для Telegram
• Вырезка · смена скорости · вшивка субтитров

Отправь ссылку на видео — скачаю!
```

**"Конвертация" card:**
```
🔄 Конвертация

• DOCX → PDF
• Ресайз и смена формата картинок
• Пришли файл — конвертирую в нужный формат

Просто отправь файл!
```

**Psychology:** Information architecture principle -- categorize, don't list. Instead of one massive message, features are organized into mental models the user already has (music / video / conversion). Each card ends with a specific CTA, not a generic one. The "1000+" number in the header creates authority through scale (anchoring bias). This approach also enables A/B testing per category -- you can track which button users tap most to understand their primary use case.

## Recommendation

**For immediate implementation:** Variant 9 (progressive disclosure with buttons).

Reasons:
1. Lowest cognitive load -- 2 lines of text
2. Leverages Telegram's native inline buttons (high tap rate, familiar UX)
3. The existing `menu.services_text` already contains the full feature list -- no new content needed for the expanded view
4. Respects both power users (they already know, just paste link) and new users (they can explore via buttons)
5. Easy to A/B test against the current version

**Runner-up:** Variant 7 (casual friend tone) for the personality factor, which builds stronger brand differentiation in a crowded bot market.

**Suggested A/B test:** Run Variant 9 vs Variant 7 and measure:
- Time to first link sent (activation speed)
- /settings usage rate (exploration depth)
- 7-day retention (personality vs minimalism)
